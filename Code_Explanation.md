# HeartSense AI - Complete Code Structure & Explanation

This document explains the organization and purpose of the codebase for the HeartSense AI project. **No code has been modified**; this is simply an explanatory guide.

---

## 1. Directory Structure Overview
The project is divided into two main folders representing the decoupled architecture:
- `/back/`: Contains the Python Flask backend server and Machine Learning logic.
- `/frontend/`: Contains the React.js client-side application.

---

## 2. Backend Code Explanation (`/back/`)
The backend is responsible for API routing, database connection, user authentication, and executing the ML model.

### Key Files & Directories:
- **`app.py`**
  This is the main entry point of the server. 
  - It initializes the Flask application and configures extensions like `Flask-CORS` (to allow frontend communication), `PyMongo` (for MongoDB access), and `JWTManager` (for secure authentication).
  - It loads the pre-trained Machine Learning model (`.pkl` file) into memory.
  - It defines all API routes (endpoints):
    - `/register` & `/login`: Handles user creation and authentication using `bcrypt` for password hashing.
    - `/predict`: Accepts patient data, formats it into an array, passes it to the Random Forest model for prediction, saves the result to the database, and returns the risk level.
    - `/chat`: Integrates with the external Groq API (LLaMA model) to act as an AI health assistant.
    - `/me`, `/history`: Fetches user profile data and past predictions.

- **`/Model/`** (or `.pkl` files in root)
  Contains the serialized Machine Learning models (e.g., `heart_disease_model.pkl`). This file is generated by training the algorithm on the dataset and saving its mathematical state so the backend can use it directly.

- **`requirements.txt`**
  Lists all the Python dependencies (Flask, scikit-learn, pymongo, etc.) needed to run the backend. This is used by Render during deployment to install the environment.

- **`test_model.py` & `test_mongo.py`**
  Unit testing scripts used during development to verify that the ML model predicts correctly and that the backend connects to MongoDB properly.

---

## 3. Frontend Code Explanation (`/frontend/`)
The frontend is a Single Page Application (SPA) built with React and Vite. It focuses on the user interface, routing, and making API calls to the backend.

### Key Files & Directories:

#### Root Configuration Files:
- **`package.json`**: Lists Node.js dependencies (React, Tailwind, Axios, Recharts, React Router) and defines scripts (`npm run dev`, `npm run build`).
- **`vite.config.js`**: Configuration for Vite, the ultra-fast build tool and development server used instead of Create React App.
- **`tailwind.config.js`**: Configuration file for Tailwind CSS, defining custom colors, fonts, and styling utility classes.

#### Source Folder (`/src/`):
This folder contains the actual application code.

- **`main.jsx` & `App.jsx`**
  - `main.jsx` is the entry point that renders the React app into the DOM.
  - `App.jsx` sets up the global structure and routing using `react-router-dom`. It maps URLs (like `/dashboard` or `/login`) to their specific page components.

- **`api.js`**
  A dedicated file for configuring `axios`. It sets the base URL for the backend API and implements an "interceptor". The interceptor automatically attaches the user's JWT token to the headers of every outgoing request, ensuring smooth authentication without writing repetitive code.

- **`/pages/` Directory**
  Contains the main views (Pages) of the application:
  - `Landing.jsx` / `Home.jsx`: The public-facing welcome pages.
  - `Login.jsx` & `Register.jsx`: Pages with forms to handle user authentication.
  - `Dashboard.jsx`: The main user dashboard showing high-level stats and links to other features.
  - `Prediction.jsx`: The core page containing the comprehensive form where users enter their 13 clinical parameters. It handles form submission, sends data to the backend, and displays the risk results using charts.
  - `PredictionHistory.jsx`: Fetches and displays a table/grid of the user's past predictions from the database.
  - `Chatbot.jsx`: The interface for the AI health assistant, displaying a chat UI and sending user messages to the backend `/chat` endpoint.

- **`/components/` Directory**
  Contains reusable UI elements:
  - `Navbar.jsx` / `Sidebar.jsx`: Navigation elements used across different pages.
  - `ProtectedRoute.jsx`: A crucial security component that wraps around private pages (like the Dashboard). It checks if a user is logged in (has a valid JWT); if not, it redirects them to the login page.
  - `RiskMeter.jsx` / `ProbabilityChart.jsx`: Visualization components used to graphically display the prediction results.

- **`/context/` Directory**
  Contains the `AuthContext`. This uses React's Context API to manage the global authentication state (is the user logged in? what is their user data?). This prevents the need to pass user data manually down to every single component.

---

## 4. Summary of their Interaction
1. A user interacts with a component in `frontend/src/pages/Prediction.jsx`.
2. React triggers a function that uses `frontend/src/api.js` to send a network request to `http://localhost:5000/predict`.
3. The Python code in `back/app.py` listens on `/predict`, receives the data, checks authentication, and executes the ML model.
4. Python queries MongoDB, saves the record, and sends a JSON response back.
5. `Prediction.jsx` receives the response and updates the screen using components like `RiskMeter.jsx`.
